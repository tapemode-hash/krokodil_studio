## Роли и доступ

При открытии сайта появляется экран входа с выбором роли:

| Роль | Пароль | Доступные разделы |
|------|--------|-------------------|
| **Гость** | не требуется | Дашборд, Расписание, Педагоги, Посещение (просмотр), Галерея |
| **Родитель** | номер телефона из базы | Те же + информация о своём ребёнке в разделе Посещение |
| **Администратор** | пароль (SHA-256) | Все разделы, включая редактирование данных |

Родитель входит, вводя свой номер телефона — система сопоставляет его с базой учеников и показывает данные только своих детей.

Пароль администратора хранится в коде в виде SHA-256-хэша (`ADMIN_PW_H`).

## Разделы приложения

### Дашборд
Главный экран. Отображается по-разному в зависимости от роли:
- **Гость / Родитель** — расписание на сегодня, объявления, контакты студии
- **Администратор** — сводная статистика: количество учеников, занятий, педагогов, выручка; расписание на сегодня, последние события

### Расписание
Список групп с расписанием занятий. Для каждой группы: название, педагог, кабинет, дни и время, статус (Активна / Набор / Пауза / Завершена).  
Администратор может добавлять и редактировать занятия.

### Ученики *(только администратор)*
Полный список учеников с поиском и фильтрацией. Карточка ученика содержит:
- ФИО, дата рождения, возраст
- Контакт родителя (имя, телефон)
- Направление (Акварель, Масло, Графика, Лепка, Батик)
- Записи на занятия (по расписанию и по датам)
- Абонементы (4 / 8 / 12 / 16 / 20 занятий)
- Стоимость занятия (индивидуальная или по умолчанию)

### Педагоги
Карточки преподавателей: имя, специализация, количество групп и учеников, рейтинг.  
Любой пользователь может поставить оценку педагогу (1–5 звёзд) и оставить отзыв.  
Администратор может добавлять и редактировать педагогов.

### Посещение
Отметка посещаемости по дате. Администратор видит все группы и может редактировать посещаемость и стоимость каждого занятия. Родитель видит только своих детей.

### Галерея
Фотографии работ учеников. Администратор может добавлять фото с описанием и тегом ученика.

### Выставки *(только администратор)*
Управление выставками работ: название, дата, описание, участники.

### Материалы *(только администратор)*
Учёт расходных материалов: заказы, остатки, поставщики.

### Оплаты *(только администратор)*
Журнал оплат с сортировкой. Привязаны к посещаемости учеников. Показывает задолженности и статус оплаты.

### Отчёты *(только администратор)*
Аналитика за период: выручка, посещаемость, загрузка педагогов. Поддерживается печать отчёта (`window.print()`).

## Хранение данных

Данные хранятся в `localStorage` с префиксом `ks_` (константа `LS`):

| Ключ | Содержимое |
|------|-----------|
| `ks_teachers` | Массив педагогов |
| `ks_students` | Массив учеников |
| `ks_schedule` | Массив групп/занятий |
| `ks_attendance` | Объект посещаемости (ключ — дата) |
| `ks_enrollments` | Записи учеников в группы |
| `ks_dateEnrollments` | Записи на конкретные даты |

При первом запуске (или сбросе) данные загружаются из встроенных констант `DEFAULT_TEACHERS`, `DEFAULT_STUDENTS`, `DEFAULT_SCHEDULE`.

**Важно:** данные хранятся только в браузере пользователя. Синхронизация между устройствами отсутствует. При очистке браузера данные будут сброшены к значениям по умолчанию.

## Локальный запуск

Так как приложение — статический HTML-файл, достаточно открыть его в браузере:

```bash
# Вариант 1: открыть файл напрямую
open index.html

# Вариант 2: локальный сервер (рекомендуется — нужен для crypto.subtle)
python3 -m http.server 8000
# затем открыть http://localhost:8000
```

> `crypto.subtle` (используется для SHA-256 при входе администратора) требует HTTPS или `localhost`. При открытии через `file://` вход администратора может не работать.
